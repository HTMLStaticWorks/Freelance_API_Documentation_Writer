A collection of documentation styles and formats we provide for our clients. Simple, clear, and developer-focused.

01. REST API Reference

GET

Retrieve Account Balance

GET /v1/account/balance

Returns the current balance and transaction history for the authenticated user.

Query Parameters
NameTypeDescription
currencystringThe currency code (e.g. USD).
Response Body
{
  "balance": 1250.75,
  "currency": "USD"
}

02. SDK Implementation

Clean code examples for various languages to help developers get started in minutes.

Node.js Python Go 
const forge = require('apiforge-node')('your_api_key');

async function getBalance() {
  const balance = await forge.account.getBalance({
    currency: 'USD'
  });
  console.log(balance);
}

03. Webhook Events

Structured event payloads for real-time data synchronization.

payment.succeeded

Sent when a payment is successfully processed and confirmed.

  • Includes transaction ID
  • Customer metadata included
  • Timestamp of completion
Webhook Payload
{
  "id": "evt_12345",
  "type": "payment.succeeded",
  "data": {
    "amount": 5000,
    "status": "paid"
  }
}

04. Authentication Flow

API Key Auth

The simplest way to authenticate. Pass your key in the Authorization header.

Authorization: Bearer sk_test_123...
OAuth 2.0 Flow

For third-party integrations, we support the standard OAuth 2.0 authorization code grant.

  • 1. Redirect to Authorization URL
  • 2. User grants access
  • 3. Exchange code for Token
